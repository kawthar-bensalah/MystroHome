Scenarios
In the section you can see all the scenarios that you have already created, their type and their status (running or stopped).

Add new scenario
To add a new scenario, click on the button. Then enter the name of your scenario. Once the name has been entered, click on the button. Then, choose the trigger mode for your scenario :

  • On demand : the scenario is triggered at any time desired by the user.
  • Programmable : the scenario will run once or at regular intervals. for example: every two hours, every Tuesday and Thursday or on certain dates.
  • Extended : the scenario can contain activities that will be triggered on different dates.

To create a simple activity, press the button circled below. The form appears you only have to enter the necessary informations : name, object and the operation the object should execute. Specify the time when the activity will be triggered. You have three types of time constraints :

  • At : the activity is triggered on a date (or a time) given by the user.
  • After : activity is triggered after another activity. In this case, you can define a timeout between the two activities.
  • Independent activity : The activity is triggered directly after the execution of the process. This option is available only for the On demand trigger mode.

To generate the condition, use “Condition Builder”. The condition has the form :
  • simple : [not] attribute operator value.
    Example : AlarmState = 'ON'
  • composed : [not ] attribute operator value [ (and | or ) [not] attribute operator value ] .
    Example :TVState = 'ON' and (TVChannel != 'TV1' or TVChannel != 'TV2')
Trigger Mode
Only for programmable trigger mode. To indicate that the scenario will be running every hour or day (or more than one hour or day), select the “Loop” option and specify the number of hours or days. The “Days” option allows you to choose the trigger days of the scenario and last the “Dates” option to specify the trigger dates. Once completed, click “Next”.
